# Trip Sheet (branded, web + PDF) — design

**Date:** 2026-06-18
**Status:** Approved (design)

## Problem

Exjet's trip sheets are generated by LevelFlight and look generic (sub-project #2 of
the quote/trip-sheet revamp). The quote redesign is shipped; the trip sheet reuses
that foundation. A trip sheet is the operational/client document for a whole trip
(dispatch): crew, per-leg departure/arrival, FBOs, weather, passengers, client block.

## Goal

A branded **Midnight** trip sheet for a dispatch, matching the current sheet's content
but redesigned, with the branded flight-path map: delivered as a **public web page**
(`/tripsheet/:id`, interactive), a **PDF**, and reachable from the **flight detail
page**. Reuses the quote infrastructure (map+plane, PDF renderer, public-route
pattern, Midnight styling).

## Decisions (resolved during brainstorming)
- **Weather:** Open‑Meteo (free, no API key) daily forecast (high/low + condition) by
  airport lat/lng.
- **Delivery:** same as the quote — public `/tripsheet/:id` (+ `/tripsheet/:id/pdf`),
  PDF, dashboard.
- **Entry point:** a "Trip Sheet" action on the **flight detail page**
  (`FlightDetail.jsx`); the leg's `dispatch._id` selects the trip.

## Data sources (from getTripLog + swagger, verified live)
`getTripLog(dispatchId)` returns `{ dispatch, aircraft, attendants, ... }` with FULL
`dispatch.legs`. Per leg: `departure/arrival` (`airport`, `time`, `fbo` {name,address,
phones}), `_calc` (`distance.value`, `time` (EFT), `from/to.location` {lat,lng}),
`passengerCount`, `pilots[]`, `attendants[]`, `passengers[]`. Dispatch: `tripId`,
`quoteId`, `client` {`company`,`customer`} (company `_id` → `/api/company-get/{id}` for
address). Weather is NOT in LevelFlight → Open‑Meteo by airport lat/lng.

## Architecture (mirrors the quote feature)

### 1. Shared map helper (small refactor)
Extract `mapScript(viewModel)` (the Leaflet route + looping plane) from
`services/quoteHtml.js` into `services/docMap.js` (exported), and import it in both
`quoteHtml.js` and the new `tripSheetHtml.js`. No behavior change to the quote.

### 2. Weather service
`services/weather.js` — `getDailyForecast(lat, lng, days = 4)` calls Open‑Meteo
(`/v1/forecast?latitude=&longitude=&daily=temperature_2m_max,temperature_2m_min,weathercode&temperature_unit=fahrenheit&forecast_days=N&timezone=auto`),
returns `[{ date, highF, lowF, condition }]`. A pure `weatherCodeLabel(code)` maps WMO
codes → text (e.g. 0→Clear, 61→Rain, 95→Thunderstorms). Soft-fails to `[]` on error.
In-memory cache per `lat,lng` (1h).

### 3. Trip-sheet data builder
`services/tripSheetData.js` — `buildTripSheet(dispatchId)`:
- `getTripLog(dispatchId)` → dispatch + legs.
- `legs`: map each full leg to `{ from, to, depTime, arrTime, distance, eft, pax,
  fromLatLng, toLatLng, fbo: { name, address, phone }, crew: { pic, sic, ca } }`.
  Crew from `leg.pilots[]` (PIC = first, SIC = second — confirm ordering/role field
  live) and `leg.attendants[]` (CA), names `firstName + lastName`.
- `client`: `{ name, company, address }` from `dispatch.client.customer`/`company` +
  `company-get` for the address.
- `tripNumber` (`dispatch.tripId`), `quoteNumber` (`dispatch.quoteId`), aircraft
  (tail, type, paxSeats).
- `weather`: for each unique airport with coords, `getDailyForecast(lat,lng)`.
- Returns the view-model the renderer consumes. Pure mapping (`mapTripLeg`) split out
  and unit-tested; the I/O (getTripLog, company-get, weather) stays in `buildTripSheet`.

### 4. Renderer
`services/tripSheetHtml.js` — `renderTripSheetHtml(vm, { print, web })`: Midnight HTML
mirroring the quote doc's styling, using the shared `docMap.js` for the map. Sections:
header (logo, **Trip #**, **Quote #**, **Prepared for** client + company + address),
aircraft card, **per-leg blocks** (crew PIC/SIC/CA, departure & arrival airport/city/
time, EFT/distance, **FBO** name+address+phone, **weather** high/low+condition),
**passengers**, flight-path map. `web` → Download-PDF action bar (`vm.pdfUrl`); `print`
→ same expand/page-break conventions as the quote.

### 5. Public + dashboard delivery
- `routes/publicTripSheets.js` (new): `GET /tripsheet/:id` (web HTML) and
  `GET /tripsheet/:id/pdf` (via the existing `renderQuotePdf(html)` — HTML-agnostic,
  reused). Mounted in `index.js` **outside** the `/api` guard (like `/quote`).
- **FlightDetail** (`frontend/src/pages/FlightDetail.jsx`): a "Trip Sheet" action
  area using the leg's dispatch id (`leg.dispatch?._id?.$oid` or `leg.dispatch?._id`):
  **View** (open `${API_BASE}/tripsheet/<dispatchId>`), **Download PDF**, **Copy link**.
  If the leg has no dispatch id, hide/disable with a note.

## Data flow
Open a flight → FlightDetail → click **View trip sheet** → `/tripsheet/<dispatchId>`
(public) renders the branded sheet (crew, FBOs, weather, animated route map) →
Download PDF or share the link.

## Edge cases
- **Missing crew/FBO/coords/weather** → those fields render blank / "—"; the layout
  holds (same graceful-degradation as the quote map).
- **Open‑Meteo down / no coords** → weather section shows "Forecast unavailable"; the
  rest renders.
- **Unknown dispatch id** → 404 on both `/tripsheet/:id` and `/pdf`.
- **Leg with no `dispatch._id`** (FlightDetail from router state) → the Trip Sheet
  action is disabled with a tooltip.
- **Public/unauthenticated** by design (dispatch id is the access token, same as the
  quote).

## Testing
- Pure: `mapTripLeg` (leg → trip-sheet leg incl. crew/fbo) and `weatherCodeLabel`
  (`node:test`).
- `node --check` on new/changed backend files; existing suites still pass.
- Live: `buildTripSheet` against a real dispatch → render web HTML + PDF; confirm
  crew/FBO/weather/map populate. Manual: FlightDetail "View trip sheet" opens the
  public page (logged out); Download PDF works.
